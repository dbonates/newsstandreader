    Quick start guide

    Application structure

    -- RootViewController - is just container for two main views, which implementation you can find in project.
-- MagazineRack/ - implementation of different components of UICollection view
-- Model
---- Asset - handles work with downloading staff like images and pdfs. You need that for downloading cover, previews and pdfs.
---- Issue - logic to work with magazine issues. It has not custom logic and just CoreData model for mapping objects from database
-- External
---- MBProgressHUD - shows progress bar when downloading issue
---- JSONKit - library to easily work with JSON answers from API calls
---- MagicalRecord - is a wrap on CoreData
---- PDFReader - simple free (under MIT) pdf reader

    Setup

    1. Change Bundle Identifier in Target
    2. Change @"password" in verifySubscription method in InAppStore.m
    3. Take a look at Reader-Prefix.pch file and define constants FREE_APP & USE_SANDBOX
    4. If you are "Developer license" holder:
      1. Change *urlPush in AppDelegate.m
      2. Change *UpdateURL & *ReceiptURL in IssueManager.m

    Customization

    Application have only two views you can customize from box:

    • MagazineRackViewController
    • MagazineDetailViewController
    and for most options you don't need more if you wanna simple newsstand application.
    Our main effort was to make customization easy and that's why we used StoryBoard. First, take a look there.

    In iOS 6 Apple has introduced a new class called UICollectionView that makes adding your own custom layouts and layout transitions (like those in the Photos app) incredibly simple to build. So MagazineRack is pretty much implementation of UICollectionView.

    Here are some usefull links if you are not familiar with UICollectionView:

    • Apple UICollectionViewController reference
    • Beginning UICollectionView. Good Tutorial by Brandon Trebitowski. You can find second part of tutorial there.
